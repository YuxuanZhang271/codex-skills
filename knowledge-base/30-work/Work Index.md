## Management Rule

When an idea becomes work, create one new Markdown note directly under `30-work/` with the same title as the source idea. Use the note to store the GitHub repo link, concrete plan, progress, and status. Do not store code or project files here; those belong in a dedicated GitHub repository.

## Promotion Rule

Promote finished work into [[40-output/Output Index|Output]] only when it produces a paper PDF or another explicitly requested output artifact.
