# Codex Knowledge Policy

The user has given Codex permission to manage the knowledge base in this repo for academic use.

## Operating Rules

- Use `knowledge-base/` as the repo-local Obsidian vault.
- Prefer Obsidian-flavored Markdown with frontmatter, wikilinks, and concise durable notes.
- Use Obsidian Bases for structured review of papers, projects, writing, and meetings.
- Keep credentials, tokens, API keys, and account secrets out of the vault.
- Preserve exact source wording when the note is intended to support applications, papers, or formal writing.
- Keep temporary task scratch separate from durable academic notes.

## Preferred Knowledge Types

- Literature notes
- Research concept notes
- Project and experiment notes
- Writing and application materials
- Meeting notes and action items
- Stable workflow preferences
